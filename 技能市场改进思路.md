# Skill 市场方案

> 参考 https://www.skillhub.cn/ ,做一个管理本地 Claude skills 的 Web 市场。
> 本地已有一系列 skills + 一台服务器 + GitLab 仓库 + 未来要新增的 skills。

## 核心原则

- **数据库只存索引,不存 skill 内容**。skill 真理源永远是 GitLab 仓库,数据库是它的投影,避免双写不一致。
- skill 本质是 `~/.claude/skills/<name>/SKILL.md`,带 YAML frontmatter 的纯文本,Git 天然友好。

## 数据流

```
                          ┌─────────────────────┐
   GitLab skill 仓库 ─────│  (真理源,存 .md)    │
   (单仓多目录 or 多仓)   └──────────┬──────────┘
                                      │ push / 定时
                          ┌───────────▼───────────┐
                          │  Indexer 服务          │
                          │  1. 拉 SKILL.md        │
                          │  2. 解析 frontmatter   │
                          │  3. 写 SQLite 索引     │
                          └───────────┬───────────┘
                                      │
            ┌─────────────────────────┼─────────────────────────┐
            ▼                         ▼                         ▼
     ┌──────────────┐         ┌──────────────┐         ┌──────────────┐
     │ Web 市场     │         │ FTS5 搜索    │         │ CLI 安装     │
     │ 浏览/筛选/详情│         │ 跨仓全文     │         │ 从 GitLab 拉 │
     └──────────────┘         └──────────────┘         └──────────────┘
```

## 数据模型

```prisma
model Skill {
  id          String   @id @default(cuid())
  name        String   @unique          // frontmatter.name
  version     String                    // frontmatter.version
  description String                    // frontmatter.description
  triggers    String[]                  // frontmatter.trigger
  tags        String[]                  // frontmatter.tags
  billing     Billing  @default(FREE)   // 计费分类,核心字段
  price       Decimal?                  // billing=PAID 时填,单位元

  // 指向 GitLab 的引用(skill 内容不在这里)
  gitlabUrl   String                    // 仓库 web 地址
  repoPath    String                    // 仓库内路径,如 skills/git-commit
  ref         String   @default("main") // 分支/tag
  sha         String                    // 最近索引的 commit,用于增量

  readme      String                    // 渲染后的 SKILL.md 正文(便于详情页展示)

  // 用户行为数据(Git 里没有,必须数据库)
  installCount Int     @default(0)
  rating       Float   @default(0)
  ratingCount  Int     @default(0)

  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  @@index([billing, tags])
  @@index([name])
}

enum Billing {
  FREE
  PAID
  FREEMIUM   // 免费+付费版
}

// FTS5 虚拟表(Prisma 不直接管,用 raw migration 建虚表)
// CREATE VIRTUAL TABLE skills_fts USING fts5(
//   name, description, tags, content='Skill', content_rowid='rowid'
// );
```

## 分阶段实施

### M1 — 索引核心:让数据进数据库

目标:能把 GitLab 的 skill 同步进 SQLite。

1. Prisma schema + SQLite + FTS5 虚拟表(migration)
2. GitLab API 客户端(列仓库 / 列目录 / 读文件)
3. frontmatter 解析器(gray-matter 库)
4. 全量同步脚本:`pnpm sync` 扫所有仓库 → 解析 → upsert
5. webhook 接收端点:push 事件 → 增量同步对应 skill
6. 用 `sha` 字段做增量,commit 没变就跳过

**验证**:改一个 SKILL.md push,GitLab webhook 触发,数据库里对应行 `updatedAt` 变化,`readme` 内容更新。

### M2 — Web 浏览(只读市场)

目标:能浏览、筛选、搜索、看详情。

1. 列表页:分页 + 按 `billing` 筛选(FREE/PAID/FREEMIUM tab)+ 按 tags 筛选
2. 搜索框:FTS5 全文搜索(name/description/tags)
3. 详情页:渲染 `readme` + 显示元数据 + 安装命令框
4. 每个详情页展示「免费/付费」徽章 + 价格(如有)

**验证**:列表页能按计费类型切换,搜索「git」能召回相关 skill,详情页 README 正常渲染。

### M3 — 安装链路

目标:用户能从市场装到本地。

最小版:详情页给安装命令,用户复制执行。
```
npx @your/skillhub-cli install <name>
```

CLI 做的事:
1. 查市场 API 拿 `gitlabUrl` + `repoPath` + `ref`
2. `git clone --depth 1` 或 sparse-checkout 只拉目标目录
3. 复制到 `~/.claude/skills/<name>/`
4. 写本地清单 `~/.claude/skillhub.json` 记录已装 skill + 版本,供 `update` 用

命令:`install <name>` / `update [name]` / `remove <name>` / `list`

**验证**:在空机器上 `install git-commit`,`~/.claude/skills/git-commit/SKILL.md` 存在且 Claude Code 能识别。

### M4 — 增强(按需)

- 安装数统计:CLI 安装时 POST 回调 +1(注意防刷)
- 评分:需鉴权,可借 GitLab OAuth
- 定时全量同步:cron 兜底 webhook 漏接
- 管理后台:审核新提交的 skill

## 技术栈

| 层 | 选型 |
|---|---|
| Web + API | Next.js (App Router) + TypeScript |
| UI | Tailwind + shadcn/ui |
| DB | SQLite + Prisma(生产可迁 Postgres,改一行连接串) |
| 搜索 | SQLite FTS5 |
| GitLab 集成 | `@gitbeaker/rest` 或直接 fetch GitLab API |
| frontmatter 解析 | `gray-matter` |
| Markdown 渲染 | `react-markdown` + `remark-gfm` |
| CLI | 同仓库单独 package,`commander` + `simple-git` |
| 部署 | Docker Compose 单机 |

## 开工前需确认

1. **skill 仓库组织方式**:
   - (a) 单仓库多目录(monorepo,如 `skills/git-commit/`、`skills/deep-research/`)
   - (b) 多个独立仓库,每个仓库一个或几个 skill
   - (c) 混合

   决定 GitLab API 调用方式(列目录 vs 列 group 下仓库)和 webhook 配置数量。

2. **frontmatter 规范是否统一**:现有 skill 的 SKILL.md 是否都带 `name/version/description/billing/tags`?如果不齐,M1 要先补规范或写个兼容层。
